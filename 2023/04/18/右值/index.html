C++的移动语义和完美转发总结

右值是什么

一般而言，一个左值表达式表示的是一个对象的身份（在内存中的位置），而一个右值表达式表示的是对象的值（内容）。从生命周期上看，左值在表达式结束后依然存在，而右值是表达式结束后就不再存在的临时对象。

根据上边的内容，区分左值和右值最简单的方法是：有名字的对象、能取地址的是左值；匿名对象、不能取地址的为右值

右值又能分为纯右值（prvalue）和将亡值（xvalue）

  • 纯右值：纯右值是下列表达式之一：对该表达式求值能够（1）计算某个运算符的操作数的值；（2）初始化某个对象的值。比如，字面量、lambda表达式、返回的临时变量等
  • 将亡值：代表它的资源能够被重新使用的对象或位域的泛左值。通常与右值引用相关，指将要被移动的对象，比如std::move返回值、返回类型为右值引用的函数返回值等。

参考：Value categories - cppreference.com

右值引用是什么

引用就是对变量的别名，右值引用就是对右值起一个别名

可以看到，普通的引用要求绑定到一个左值上，如果用int& ri = 0;这样的写法会报错，而右值引用则可以。可绑定类型的值总结一下：

  • 左值引用：只能绑定到非const左值（如果允许绑定常量左值和常量右值，意味着可以修改被绑定的值，与const定义不符）
  • 右值引用：只能绑定到非const右值
  • 常量左值引用：这个是万能的引用，既可以绑定到所有类型的值

之前提到过，右值在表达式结束后就被销毁，但是右值引用会给被绑定到的右值延长生命周期，变得与右值引用相同。

万能引用

概念：使用T&&类型的形参既能绑定右值，又能绑定左值。

前边不是说右值引用只能绑定到非const右值吗，为什么这里能绑定左值？因为之前的情况中T是一个具体的类型，T&&确定为右值引用。在发生类型推导的时候，T&&表示万能引用（universal reference）

万能引用T&&的类型推导规则，以int为例，当传入实参为左值时，T被推导为int& ;当传入实参为右值时，T被推导为非引用类型int。

If a variable or parameter is declared to have type T&& for some deduced type T, that variable or parameter is a universal reference.

Universal References in C++11 -- Scott Meyers : Standard C++ (isocpp.org)

引用折叠

万能引用之所以万能，在于使用模板推导和引用折叠的规则，生成不同的实例化模板接收传进来的参数

引用折叠的核心是，正常情况下C++不允许定义引用的引用，但C++编译器在某些情境下（模板实例化、auto类型推导、创建和运用typedef和别名声明、decltype）有可能会定义引用的引用这种形式，这时会形成引用折叠。

  • 所有其他引用类型之间的折叠（T& &，T& &&，T&& &）都是左值引用

移动语义

如果类中有动态分配的内存，则必须使用深拷贝进行拷贝构造，否则可能出现空悬指针的问题（A和B中两个指针指向同一内存，B析构释放了该内存，但A仍然存在）

问题在于，深拷贝有些时候会造成不必要的性能开销，比如：有个函数func，返回值为B，然后要用B去构造A，之后B就不会再使用了。这时候还要用B去做深拷贝？为啥不能直接继续用B？

所以很自然的联想到右值引用延长生命周期的作用，就有了专门的移动构造函数去接收右值引用参数。移动构造参数避免深拷贝的做法是将指针转移到被构造对象上，然后对参数对象的指针置空，避免了空悬指针的出现

上面是用右值作为参数去构造的，如果有个左值，后面不会再使用了，能不能也用类似的方式拿来构造一个新对象？

如果传入warp的是左值，std::forward接受的是一个左值引用，这里_Ty被推导为int&类型，被实例化为接受和返回左值引用。内部的转换不做任何事。

如果传入warp的是右值，std::forward接受的是一个右值引用，这里_Ty被推导为int类型，被实例化为接受和返回右值引用。内部的转换不做任何事。

MSVC这个实现里有个注释”forward an lvalue as either an lvalue or an rvalue“，这个是对传入的是左值（forward传入的是左值，不是引用类型）说的，如果传入的是左值，_Ty被推导为int，那个最后返回的类型是int&&,这个作用就相当于std::move()（但是真的会有人这么用吗……）

移动vs拷贝

之前提到移动是为了避免不必要的深拷贝造成的时间开销，从而提高性能，那移动一定比拷贝要快吗？

不一定，移动快的原因是直接把指针的所有权转移，如果数据类型没有指针，比如基础类型，或者STL中的array，移动和拷贝是等价的。都还是要对元素进行”搬家“。
